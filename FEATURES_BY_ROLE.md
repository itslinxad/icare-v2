# iCARE++ System Features by User Role

This document outlines the features and functionalities available to each user role in the iCARE++ system, as documented in the manuscript.

---

## Administrator (Dean)

The system administrator, designated as the Dean, is responsible for institutional-level management and oversight.

| Feature | Description |
|---------|-------------|
| **User Account Provisioning** | Create, manage, and maintain user accounts across all roles |
| **Role-Based Access Control (RBAC) Configuration** | Configure and manage access permissions for nursing students, faculty, and administrators |
| **Data Management** | Centralized data management within the institution |
| **Cohort-Level Analytics** | View analytics across entire student cohorts |
| **At-Risk Student Reports** | Generate and review reports identifying students at academic or clinical risk |
| **Comprehensive Audit Log** | Access complete audit trail of all system activities and user interactions |
| **Multi-Campus Support** | Manage and scale deployment across multiple campuses |
| **Institutional Analytics Dashboard** | Star schema data warehouse with consolidated analytics |
| **Compliance Management** | Ensure compliance with the Philippine Data Privacy Act of 2012 |

---

## Faculty (Head Nurse)

Faculty members serve as supervising head nurses and monitor student performance within the simulation environment.

| Feature | Description |
|---------|-------------|
| **ML-Powered Analytics Dashboard** | Real-time access to student performance data with machine learning insights |
| **Competency Report Review** | View and generate competency reports for students |
| **Simulation Scenario Management** | Create, manage, and oversee clinical simulation scenarios |
| **At-Risk Student Identification** | Automatic identification and flagging of at-risk students |
| **Audit Trail Access** | Complete audit trail of all clinical activities and student interactions |
| **Student Performance Monitoring** | Monitor individual student progress in real-time |
| **Post-Simulation Debriefing Tools** | Data-driven tools for meaningful post-simulation feedback |
| **Real-Time Notifications** | Receive alerts and notifications about student activities |
| **Clinical Decision Support** | AI-driven clinical decision support module for simulated patient case management |

---

## Student

Nursing students are the primary users of the system, interacting with it as a clinical training and healthcare management platform.

| Feature | Description |
|---------|-------------|
| **Vital Signs Monitoring** | Monitor patient vital signs with rule-based anomaly detection |
| **Adaptive Quiz System** | Answer adaptive quiz questions personalized to knowledge gaps |
| **Clinical Task Completion** | Complete assigned clinical activities |
| **Simulated EHR Environment** | Interact with simulated Electronic Health Records |
| **Patient Case Management** | Practice clinical documentation and patient management |
| **Clinical Decision Support** | Receive evidence-based clinical guidance and alerts |
| **AI-Driven Recommendations** | Personalized learning recommendations based on performance |
| **Real-Time Notifications** | Receive alerts about assignments, deadlines, and performance |
| **Mobile Application Access** | Access via dedicated mobile app (Android and iOS) |
| **Offline Functionality** | PWA with offline-first architecture for low-connectivity environments |
| **Progress Tracking** | Track individual performance and competency development |

---

## Common System Features

The following features are available across all user roles:

| Feature | Description |
|---------|-------------|
| **Machine Learning Performance Prediction** | Uses Random Forest and Logistic Regression algorithms |
| **Star Schema Data Warehouse** | Centralized data repository for analytics |
| **Progressive Web Application (PWA)** | Cross-platform web access with offline capabilities |
| **Cross-Platform Mobile App** | React Native app for Android and iOS |
| **Role-Based Access Control** | Secure access based on user roles |
| **Audit Trail System** | Chronological record of all system activities |
| **PDF Report Generation** | Automated competency report generation |
| **Multi-Campus Scalability** | Architecture supports deployment across multiple institutions |
